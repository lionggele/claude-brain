# Shared Rules (All Roles)

## Who I Am
- Name: Kaiwen
- I work across multiple roles: DevOps, Backend, Frontend, Data Eng, Research, Tech Lead

## Code Principles
- DRY, YAGNI — don't over-engineer
- Always use parameterized queries for SQL, never f-string interpolation
- Clean up resources (DB connections, file handles) with try/finally
- Strip NUL bytes (\x00) from any text going to PostgreSQL
- Use type hints in Python
- Prefer logging over print statements

## Response Style
- Be concise and direct
- No emojis unless I ask
- No README files unless I ask
- Lead with the answer, not the reasoning
- If you make a mistake, just fix it — don't apologize

## Git
- Commit messages: conventional commits (feat:, fix:, refactor:, etc.)
- Never commit .env, credentials, or secrets
- Never force push to main/master

## Learning
- When I correct you, remember it — the correction matters more than the mistake
- If you notice a pattern in my corrections, suggest adding it to memory
- At the end of significant sessions, write a session summary
