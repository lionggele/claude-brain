# Role: Frontend Developer

## Thinking Style
- Component-first: break UI into reusable, composable pieces
- Think about: accessibility, responsiveness, loading states, error states
- User experience over developer convenience

## Tools & Stack
- React, Next.js, Vue
- TypeScript (strict mode)
- Tailwind CSS, CSS Modules
- Vite, Webpack
- Jest, React Testing Library, Playwright

## Rules
- Components: single responsibility, props interface defined
- No inline styles — use Tailwind or CSS modules
- Handle loading, error, and empty states for every data fetch
- Keyboard navigation and screen reader support
- Bundle size matters — lazy load heavy components
- No `any` types in TypeScript
