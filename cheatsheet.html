Claude Brain v1.0

Role-based, self-learning Claude Code configuration — 10 roles • 10 skills • confidence-scored memory

🔧 Install
git clone https://github.com/lionggele/claude-brain.git
cd ~/Projects/claude-brain
bash install.sh
What install.sh does
  1. Symlinks repo → ~/.claude/brain/
  2. Adds @brain import to ~/.claude/CLAUDE.md
  3. Symlinks 10 skills to ~/.claude/skills/brain-*
  4. Creates .local/ for personal data (gitignored)
  5. Creates projects/ for artifacts

After install, every Claude Code session loads brain rules + skills.

⚙️ How It Works
Auto-load chain
~/.claude/CLAUDE.md
  @~/.claude/brain/CLAUDE.md
    @shared/CLAUDE.md            # all roles
    @roles/backend/CLAUDE.md     # active role
    @shared/memory/corrections.md # learnings
Skill discovery
~/.claude/skills/brain-*/SKILL.md
  # auto-discovered as /brain-* commands
Every session
  1. Reads ~/.claude/CLAUDE.md → loads brain via @ imports
  2. Discovers brain-* skills → registers slash commands
  3. If project CLAUDE.md has context → loads artifacts

Manual capture (terminal)
# new learning
bash ~/.claude/brain/scripts/smart-capture.sh \
  --role backend --confidence 0.3 --seen 1 "Use connection pooling"
# promote to permanent rule
bash ~/.claude/brain/scripts/smart-capture.sh \
  --role backend --promote "Use connection pooling"
🔄 Session Flow
  1. Open terminal in project dir
  2. Start Claude Code: claude
  3. Brain auto-loads (rules + skills)
  4. Verify: /brain-brain-status
  5. Work — brain rules apply automatically
  6. Correct Claude → smart-capture saves it
  7. Decisions: /brain-spike → artifact
  8. End: /brain-session-summary
  9. Push: bash ~/.claude/brain/scripts/sync.sh
